# HOLLOW — ระบบจองโต๊ะบาร์

เว็บจองโต๊ะที่ลูกค้ากดเลือกโต๊ะจากแผนผังร้านได้เลย เลือกวัน–เวลา จำนวนคน
กรอกชื่อ เบอร์โทร และไอดีไอจี พร้อมหน้าแอดมินสำหรับดูและยกเลิกการจอง

เขียนด้วย **Python (ไลบรารีมาตรฐานล้วน) + SQLite** — ไม่ต้องติดตั้งแพ็กเกจใด ๆ

---

## วิธีรัน

ต้องมี Python 3.8 ขึ้นไป (มากับ macOS/Linux อยู่แล้ว; Windows โหลดที่ python.org)

```bash
cd bar-booking
python3 server.py
```

แล้วเปิดเบราว์เซอร์:

- หน้าลูกค้าจองโต๊ะ → http://localhost:8000
- หน้าแอดมิน → http://localhost:8000/admin.html (เบราว์เซอร์จะเด้งขอ user/password)

หยุดเซิร์ฟเวอร์ด้วย `Ctrl + C`

> เปลี่ยนพอร์ตได้ด้วย `PORT=9000 python3 server.py`

### รหัสผ่านหน้าแอดมิน

หน้า `/admin.html` และ API ที่แอดมินใช้ (`GET /api/reservations`, `POST /api/reservations/{id}/cancel`)
ถูกล็อกด้วย HTTP Basic Auth ตั้งค่าได้ด้วย env var:

```bash
ADMIN_USER=admin ADMIN_PASSWORD=รหัสผ่านของคุณ python3 server.py
```

ถ้าไม่ตั้ง `ADMIN_PASSWORD` ระบบจะใช้ค่าเริ่มต้น `changeme` และเตือนใน console —
**ต้องเปลี่ยนก่อนใช้งานจริงเสมอ**

## โครงสร้างไฟล์

```
bar-booking/
├── server.py          # เว็บเซิร์ฟเวอร์ + REST API + auth หน้าแอดมิน
├── db.py              # ฐานข้อมูล SQLite (สคีมา, ข้อมูลตั้งต้น, ตรรกะการจอง)
├── bar.db             # ไฟล์ฐานข้อมูล (สร้างอัตโนมัติครั้งแรกที่รัน, ไม่ต้อง commit)
├── requirements.txt   # ว่างเปล่า — บอก Railway/Nixpacks ว่าเป็นโปรเจกต์ Python
├── nixpacks.toml      # กำหนดเวอร์ชัน Python + คำสั่งสตาร์ทให้ Railway
├── railway.json       # ตั้งค่า build/deploy ของ Railway
└── public/
    ├── index.html     # หน้าลูกค้า
    ├── style.css      # ธีมบาร์กลางคืน
    ├── app.js         # แผนผังโต๊ะ + ตรรกะการจอง
    ├── admin.html     # หน้าแอดมิน
    └── admin.js
```

## ฟีเจอร์

- **แผนผังร้านกดได้** — วางตำแหน่งโต๊ะ 1–7 และเก้าอี้บาร์ 5 ตัวตามแปลนร้านจริง
  โต๊ะที่ว่างกดเลือกได้ โต๊ะที่เต็มจะเป็นสีจาง โต๊ะที่เล็กเกินจำนวนคนจะเลือกไม่ได้
- **เลือกวัน / เวลา / จำนวนคน** — ระบบเช็คที่ว่างของแต่ละช่วงเวลาให้อัตโนมัติ
- **บังคับกรอกเบอร์โทรและไอดีไอจี** — เพื่อให้ร้านติดต่อกลับและยืนยันตัวลูกค้าได้
- **กันจองซ้ำ** — ถ้ามีคนจองโต๊ะเดิมเวลาเดิมตัดหน้า ระบบจะปฏิเสธและให้เลือกใหม่
- **หน้าแอดมิน (ล็อกด้วยรหัสผ่าน)** — ดูรายการจองรายวัน สรุปยอด และกดยกเลิกได้
- **เช็คบิล** — กดเช็คบิลแล้วใส่ยอดที่ลูกค้าจ่าย โต๊ะจะว่างให้จองใหม่ได้ทันที พร้อมยอดรวมต่อวันในหน้าแอดมิน

## การปรับแต่ง

| อยากแก้ | แก้ที่ |
|---|---|
| จำนวนโต๊ะ / ความจุ / ชื่อโต๊ะ | `db.py` ตัวแปร `SEED_TABLES` |
| ช่วงเวลาที่เปิดจอง | `public/app.js` ตัวแปร `TIME_SLOTS` |
| ตำแหน่งโต๊ะบนแผนผัง | `public/app.js` ตัวแปร `LAYOUT` |
| สี / ฟอนต์ / ธีม | `public/style.css` (ตัวแปร `:root`) |

> หากแก้ `SEED_TABLES` หลังจากรันไปแล้ว ให้ลบไฟล์ `bar.db`
> แล้วรันใหม่ เพื่อให้ระบบสร้างข้อมูลตั้งต้นชุดใหม่

## API (สำหรับต่อยอด)

| Method | Path | หน้าที่ |
|---|---|---|
| GET | `/api/tables` | รายการโต๊ะทั้งหมด |
| GET | `/api/availability?date=YYYY-MM-DD&time=HH:MM` | โต๊ะที่ถูกจองแล้วในช่วงนั้น |
| POST | `/api/reservations` | สร้างการจอง |
| GET | `/api/reservations?date=YYYY-MM-DD` | รายการจอง (แอดมิน) |
| POST | `/api/reservations/{id}/cancel` | ยกเลิกการจอง |
| POST | `/api/reservations/{id}/checkout` | เช็คบิล (ใส่ยอดที่จ่าย, โต๊ะว่างให้จองใหม่ได้) |

## Deploy ขึ้น Railway (พร้อมที่เก็บข้อมูลถาวร)

โค้ดนี้เตรียมไว้ให้ deploy ได้เลยด้วยไฟล์ `nixpacks.toml` และ `railway.json`
เซิร์ฟเวอร์อ่านพอร์ตจาก env var `PORT` อยู่แล้ว (Railway ตั้งให้อัตโนมัติ)

**1. เตรียมโค้ด**

Railway ดึงโค้ดจาก GitHub repo หรืออัปโหลดตรงด้วย Railway CLI ก็ได้:

```bash
npm i -g @railway/cli
railway login
railway init
railway up
```

หรือ push ขึ้น GitHub แล้วในหน้า Railway เลือก **New Project → Deploy from GitHub repo**

**2. เพิ่ม Volume (ที่เก็บข้อมูลถาวร)**

`bar.db` เป็น SQLite ไฟล์เดียว — ถ้าไม่ mount volume ข้อมูลจะหายทุกครั้งที่ deploy ใหม่
เพราะ container ของ Railway ไม่ persist ไฟล์ระบบตามปกติ

ในหน้าโปรเจกต์ Railway → service ของแอปนี้ → แท็บ **Volumes** → **New Volume**
ตั้ง mount path เป็น `/data`

**3. ตั้ง Environment Variables**

ในแท็บ **Variables** ของ service ใส่:

| ตัวแปร | ค่า |
|---|---|
| `DB_PATH` | `/data/bar.db` |
| `ADMIN_USER` | ชื่อผู้ใช้แอดมิน (เช่น `admin`) |
| `ADMIN_PASSWORD` | รหัสผ่านที่คาดเดายาก — **ต้องตั้งเอง ห้ามใช้ค่า default** |

Railway จะตั้ง `PORT` ให้อัตโนมัติ ไม่ต้องเพิ่มเอง

**4. Deploy**

กด Deploy (หรือ `railway up` อีกครั้งถ้าใช้ CLI) แล้วเปิดโดเมนที่ Railway สร้างให้
ครั้งแรกที่รัน ระบบจะสร้างไฟล์ `bar.db` ที่ `/data/bar.db` บน volume อัตโนมัติ
และจะอยู่ถาวรข้าม deploy ครั้งต่อ ๆ ไป

## ก้าวต่อไปที่แนะนำ (ถ้าจะใช้งานจริง)

ตอนนี้เหมาะสำหรับใช้ในร้าน/เดโมได้เลย ถ้าจะเปิดออนไลน์จริงควรเพิ่ม:

- ยืนยันเบอร์โทรด้วย OTP หรือส่ง SMS/อีเมลยืนยันการจอง
- กำหนดระยะเวลานั่งต่อรอบ (เช่น กันโต๊ะ 2 ชั่วโมงต่อการจอง)
- สำรองข้อมูล (backup) ไฟล์ `bar.db` บน volume เป็นระยะ
- ถ้ามีแอดมินหลายคน ควรทำระบบผู้ใช้แยกรายคนแทน user/password ตัวเดียว
